Node.js and V8 Engine

Node.js

    Node.js is an open-source and cross-platform Javascript runtime envirionment. It executes JavaScript code outside of a web browser. Node.js can help developers use JS to write command line tools and for server-side scripting.[1]

    Node.js runs the v8 JavaScript engine outside the Googole browser. A Node.js app is run in a single process, without creating extra threads for every request.

    With time passing by, developers have built thousands of libraries upon Node.js that worthing learning. For example: Express, AdonisJs, hapi, Angular, Meteor, ect..


V8 Engine

    V8 is the name of the JAVASCRIPT engine that powers Google chrome. We can understand like this: V8 provides the runtime environment in which JAVASCRIPT executes. And the browser provides the DOM, and other web platform APIs.

     The first version of the V8 engine was released at the same time as the first version of Chrome: September 2, 2008 in order to increase the performance of JAVASCRIPT execution inside web browsers.

    Before version 5.9 of V8 came out (released earlier this year), the engine used two compilers: full-codegen and Crankshaft

    In 2016, in order to reduce the memory usage on small memory Android phones, the Ignition interpreter was added to V8.

    In 2017, V8 shipped a brand-new compiler pipeline, consisting of Ignition (the interpreter) and TurboFan (the optimizing compiler). Starting with V8 version 5.9, Full-codegen and Crankshaft are no longer used in V8 for JavaScript execution, since the team believes they are no longer able to keep pace with new JavaScript language features and the optimizations those features require.[2]


How Node utilizes V8 engine?

    Before this let's get some fundamentals about computers first. All of our computers consist of microprocessors. Microprocessors are michine components that receive instructions and electrical signals. The instructions are machine languages that the microprocessors can interpret. Most languages we use today are advanced languages and the microprocessors don't recognize. In order to make the microprocessor understand the languages we use, the browser has to convert the advanced language into machine language.

    Now let's go back to the V8 engine: V8 is a powerful open source Javascript engine provided by Google. It is a program that converts Javascript code into lower level or machine code that microprocessors can understand. This is where Node can utilize V8 engine.


Interesting fingdings

    I am curious by the different languages I encountered when I did research on V8 engine. Why there are different languages on various levels and how to use them?

    A high-level language is easy for programmers to write and understand. Programmers can use simple and easy syntax to address a specific task. Examples: Python, C, C++

    Assembly language falls between a high-level programming language and Machine language. it has syntaxes similar to English, but more difficult than high-level programming languages. To program in assembly language, one should have understood at hardware level like computer architecture, registers, etc.

    Machine language is the binary language that is easily understood by computers but difficult for programmers. Hence it can be directly executed by CPU with absolutely no need of compilers and interpreters. Machine language has a high efficiency when executed by CPU.

    In this era, few programmers will learn machine languages because it's difficult to learn and the efficiency to write is low. With the help of compiliers and interpreters, programmers can use high-level languages to write codes efficiently.
